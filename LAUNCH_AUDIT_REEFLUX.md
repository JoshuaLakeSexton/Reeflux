# Reeflux Launch Audit (GO Pass)

## What Changed
- Removed broken Mirror Pool links and replaced user-facing text with `Mirror Pool - inbound.`
- Standardized success flow to `/success` in UI copy, links, and checkout function defaults
- Fixed checkout function syntax and set Stripe Checkout `success_url` to `/success`
- Centralized pricing display in one UI config constant (`PRICING`) in `app.js`
- Replaced Drift Pass placeholder price with real value (`$5/mo`) across the UI
- Made home pool tile labels match gating reality (`pass required`)
- Enforced pass-gating on Tide Deck (added `data-pool-gate` and `data-pool-content` behavior)
- Removed visible telemetry placeholders (`--`) from Reef Status and Tide Deck surfaces
- Upgraded `/success` UX with clear next actions and refresh behavior
- Reworked Sandbox controls IA into grouped sections: Generate / Export / Scratchpad
- Renamed unclear Sandbox controls (`Prompt Roulette`, `Clear Scratchpad`) and improved 375px wrapping

## Before / After (7 Required Fixes)

### 1) Broken `/mirror-pool.html` link
- Before: Requests and Redeem pages linked to `/mirror-pool.html` (dead path)
- After: Links removed and replaced with `Mirror Pool - inbound.` text; no page links to `mirror-pool.html`

### 2) Standardize success route to `/success`
- Before: Token Booth and copy referenced `/success.html`; checkout pathing was inconsistent
- After: Token Booth now links and references `/success`; checkout function defaults success to `/success`; success redirect/copy uses `/success`

### 3) Drift Pass pricing placeholder
- Before: `Buy Drift Pass ($/mo)` placeholder appeared in UI
- After: Real price is displayed as `$5/mo`; pricing labels now resolve from one config object (`PRICING`) in `app.js`

### 4) OPEN vs SEALED contradiction
- Before: Home tiles said `open`, but pool pages could show `Pool sealed`
- After: Home tiles now say `pass required`; Tide Deck is now gated consistently like other primary pools

### 5) Placeholder telemetry (`--`)
- Before: Home Reef Status and Tide Deck displayed `--` placeholders
- After: Home and pool surfaces now use intentional launch states (`loading`, `quiet window`, `awaiting first event`) with truthful limited-mode messaging

### 6) `/success` page dead-end/confusion
- Before: Success page lacked clear escape path and guidance
- After: Success page now includes:
  - `Access set on this device.`
  - `Enter Reef (Home)` button
  - `Open Pools` button
  - `Refresh Access` button
  - fallback text: `If you arenâ€™t redirected, click Enter Reef.`

### 7) Sandbox controls IA polish
- Before: Controls were flat and mixed; unclear labels like `Wipe`
- After: Controls are grouped into Generate / Export / Scratchpad sections; labels clarified (`Prompt Roulette`, `Clear Scratchpad`); mobile layout wraps cleanly at 375px
